# whodid · 집안일 체크

오늘 한 집안일을 한 번의 탭으로 체크하고, 매일의 기록을 잔디·달력·통계로 돌아보는
**혼자서도, 가족이 함께도** 쓰는 웹 앱입니다. 데이터는 기본적으로 브라우저(localStorage)에만
저장되어 완전히 오프라인으로 동작하고, 서버·계정·로그인이 없습니다. 원하면 **iCloud Drive의
파일 하나**에 묶어 맥·아이폰 사이에서 기록을 맞출 수 있어요(우리 서버가 아니라 사용자의
iCloud를 그대로 씁니다). → [맥·iOS에서 쓰기](#맥ios에서-쓰기-설치--icloud-동기화)

## 실행

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # 프로덕션 빌드 (dist/)
npm run preview  # 빌드 결과 미리보기
npm run lint
npm test         # 순수 코어 로직 단위 테스트 (Vitest)
```

모바일(폰)에서 쓰기 좋게 480px 단일 컬럼으로 설계했지만 데스크톱에서도 잘 동작합니다.

## 기능

- **오늘**: "미리 언제 한다"를 정하는 게 아니라, **해놓고 보니 이걸 했네** 하고 탭해서 남기는
  기록이에요. 집안일 종류를 큰 행으로 보여주고, 행을 탭하면 오늘 한 것으로 체크(다시 탭하면 해제).
  가운데 링은 오늘 몇 가지를 기록했는지 세어 주고, 다 기록하면 하루 한 번 가벼운 컨페티.
  `+` 버튼으로 같은 일을 하루 여러 번 기록(×N). 하단 바에서 새 집안일 바로 추가.
  요일 반복을 걸어 두면 그날 목록이 정돈되고, 나머지는 "다른 집안일도 기록하기"로 접혀 언제든 체크할 수 있어요.
- **구성원**: 가족과 함께 쓸 때 "누가 했는지"를 기록. 오늘 화면 상단의 사람 칩으로 기록자를
  고르면 탭 체크가 그 사람에게 귀속돼요. 계정·서버 없이 이 기기에만 저장하며,
  **구성원이 0~1명이면 관련 UI는 완전히 숨겨져** 혼자 쓰는 경험은 그대로예요.
- **기록**: 1년치 잔디 히트맵(오늘이 보이도록 자동 정렬) ↔ 달력 토글. 날짜를 탭하면
  하루 상세 시트에서 그날 한 일(누가 했는지 포함)을 보고, **과거 날짜도 소급 체크/메모** 가능(미래는 비활성).
- **통계**: 이번 주·이번 달 기록 개수(지난 기간 대비 증감),
  **이번 주 구성원별 분담**, **월간 회고 카드**, **주간 추세 스파크라인**,
  가장 많이 한 일 Top 3, 요일별 활동 막대. (차트 라이브러리 없이 CSS/SVG로 구현)
- **관리**: 집안일 추가/수정(요일 반복 설정 포함), 드래그로 순서 변경, 보관(소프트 삭제)·삭제,
  보관함 복구, 가족 구성원 추가/수정/삭제.
- **설정**: 테마(시스템/라이트/다크), 진동 피드백, 정렬 모드, 삭제 확인,
  **홈 화면에 추가(PWA 설치)**, **데이터 내보내기/가져오기(JSON 백업)**, 전체 초기화.
- 자정이 지나면 "오늘"이 자동 갱신되고, 여러 탭을 열어도 서로 동기화됩니다.
- **오프라인**: 서비스워커가 앱 셸을 캐시해, 설치 후에는 네트워크 없이도 즉시 실행돼요.

## 맥·iOS에서 쓰기 (설치 + iCloud 동기화)

맥과 아이폰 양쪽에서 **실행 아이콘**으로 열고, **iCloud Drive 폴더**로 기록을 맞추는 방법입니다.

### 1. 빌드

```bash
npm run build     # dist/ 생성
```

`base: './'`(상대 경로)로 빌드되므로, 루트든 하위 경로든 iCloud Drive 폴더에서 직접 열든
**어디에 둬도** 그대로 동작해요.

### 2. 어디서 열까

- **정적 호스트(권장)** — `dist/`를 HTTPS 정적 호스트(Netlify·Vercel·Cloudflare Pages·
  GitHub Pages 등)에 올리면 맥·아이폰 **모두** 설치형 앱이 되고 오프라인도 됩니다.
  ```bash
  npx serve dist          # 로컬에서 먼저 확인
  ```
- **맥에서 iCloud Drive 폴더째로 열기** — `dist/`를 iCloud Drive 폴더(예:
  `iCloud Drive/whodid/`)에 두고 `index.html`을 브라우저로 엽니다. 맥에서는 이대로도 잘 돼요.
  (단 `file://`에선 오프라인 서비스워커가 동작하지 않고, 아이폰은 파일에서 홈 화면 설치가
  제한적이라 아이폰까지 쓰려면 위의 정적 호스트 방식을 권장합니다.)

### 3. 실행 아이콘 만들기

| 기기 | 방법 |
| --- | --- |
| 아이폰·아이패드 (Safari) | 공유 → **홈 화면에 추가** → whodid 아이콘 생성 |
| 맥 (Chrome·Edge) | 주소창 오른쪽 **설치** 아이콘 또는 메뉴 → **앱 설치** → Dock·런치패드에 등록 |
| 맥 (Safari 17+) | 파일 → **Dock에 추가** |

### 4. iCloud로 기기 간 동기화

1. iCloud Drive에 폴더를 하나 만들고(예: `whodid`), 그 안에 데이터 파일 `whodid.json`을 둡니다.
2. **맥의 Chrome·Edge** — 앱에서 **설정 → 기기 간 동기화 → iCloud 파일 연결**을 누르고 그
   `whodid.json`을 고릅니다. 이후에는 **자동으로** 읽고 씁니다(앱을 다시 열거나 창을 포커스할
   때 최신본을 당겨옵니다).
3. **그 외(맥·아이폰의 Safari 등)** — 자동 연결 API가 없으므로 **설정 → 백업 → 내보내기**로
   같은 `whodid.json`에 저장하고, 다른 기기에선 **가져오기**로 불러옵니다(두 번 탭).
4. 그 폴더를 가족과 **공유**하면 온 가족이 같은 기록을 봅니다.

동기화되는 것은 **집안일·기록·구성원**입니다. 테마·진동 같은 **기기 설정과 ‘지금 이 기기에서
기록 중인 사람’은 기기별로** 남습니다. 같은 파일이 양쪽에서 바뀌면 **더 최근 저장본**이 이깁니다.

> **솔직한 한계** — 파일 자동 연결(File System Access API)은 **맥의 Chrome·Edge에서만** 됩니다.
> iOS의 모든 브라우저와 맥 Safari에는 그 API가 없어, 아이폰에서는 내보내기/가져오기(수동, 두
> 번 탭)로 맞춥니다. 데이터는 어느 경우에도 우리 서버로 가지 않고 사용자의 iCloud/기기에만
> 머뭅니다.

## 기술

React 19 + Vite. 외부 런타임 의존성 0개(라우터·상태관리·차트·드래그 라이브러리 없음).
상태는 단일 `StoreProvider`(useReducer)에서 관리하고, 다섯 개의 네임스페이스 키
(`whodid:chores` / `whodid:logs` / `whodid:streak` / `whodid:settings` / `whodid:members`)로
localStorage에 저장합니다. 스키마 v2는 구성원(`members`)과 항목별 기록자(`entry.by`),
집안일별 요일 반복(`chore.days`)을 더하되 **기존 v1 데이터를 그대로 보존**하는 순수 추가형
마이그레이션입니다. 모든 날짜 키는 로컬 타임존 기준 `YYYY-MM-DD`이며,
읽기/쓰기는 try/catch로 보호되고 손상된 데이터는 해당 키만 기본값으로 복구합니다.
iCloud 동기화는 별도의 얕은 계층입니다 — `cloudsync`가 File System Access API로 파일 하나를
읽고 쓰고(핸들은 IndexedDB에 보관), `useCloudSync`가 상태 변화를 감지해 디바운스 후 파일에
반영하고 포커스 때 최신본을 당겨옵니다. 무엇을 채택/밀어넣기/보류할지 정하는 병합 판단
(`chooseSync`·`chooseRepull`)은 부수효과 없는 순수 함수라 단위 테스트로 고정돼 있습니다.
`reducer`·`selectors`·`date`·`migrate`·`cloudsync` 등 순수 함수는 Vitest로 고정돼 있습니다.

```
src/
  store/      StoreProvider, reducer, selectors, migrate, defaults, context (+ *.test.js)
  lib/        date, id, copy, backup, cloudsync, confetti, haptic (+ *.test.js)
  hooks/      useStore, useTheme, useMidnightRollover, useInstallPrompt, useCloudSync
  components/ common · today · history · stats · manage · settings + shell
  styles/     tokens.css(디자인 토큰) · global.css
public/       sw.js(오프라인 셸) · manifest · 아이콘
```

## 배포

정적 호스트(Netlify, Vercel, Cloudflare Pages, GitHub Pages 등) 어디든 올릴 수 있어요.
라우터가 없는 단일 페이지라 리라이트 규칙이 필요 없고, `base: './'`(상대 경로)라 **하위 경로
(`example.com/whodid/`)에 올려도 그대로** 동작합니다.

```bash
npm run build            # dist/ 생성
# dist/ 폴더를 정적 호스트에 업로드하거나,
npx serve dist           # 로컬에서 프로덕션 결과 확인
```

서비스워커·매니페스트가 함께 배포되어, HTTPS로 서비스하면 모바일에서 "홈 화면에 추가"로
앱처럼 설치되고 오프라인에서도 실행됩니다. (설정 화면에도 설치 버튼이 노출돼요.)

데이터는 기본적으로 이 기기/브라우저에만 저장됩니다. 캐시를 지우면 사라지므로,
중요한 기록은 설정 → 백업 → 내보내기로 주기적으로 백업하거나,
[iCloud 동기화](#맥ios에서-쓰기-설치--icloud-동기화)를 켜 두세요.
